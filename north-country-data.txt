Aquí va de nuevo, con foco absoluto en el fallo “desde /marketplace” → hash links se van a “/” en vez de “/#contacto”:

1️. Asegúrate de que el Router esté bien ordenado
En App.tsx pon primero la ruta "/marketplace" y luego la raíz "/":

tsx
Copiar
Editar
import { Router, Route } from 'wouter'

export default function App() {
  return (
    <Router>
      <Header />
      {/* IMPORTANTE: marketplace antes de la ruta "/" */}
      <Route path="/marketplace" component={Marketplace} />
      <Route path="/" component={Home} />
    </Router>
  )
}
Si el "/" viene primero, Wouter castea también /marketplace como home y «se traga» el hash.

2️. Usa enlaces HTML puros con recarga completa desde /marketplace
En Header.tsx, para las secciones de la home NUNCA uses <Link> de Wouter, ni onClick, ni JS extra. Esto obliga a que el navegador haga full reload y salte al ancla:

tsx
Copiar
Editar
<nav className="…">
  {/* 👉 SECCIONES HOME (recarga + scroll automático) */}
  <a href="/#inicio"    target="_self" className="px-6 py-3">Inicio</a>
  <a href="/#nosotros"  target="_self" className="px-6 py-3">Nosotros</a>
  <a href="/#servicios" target="_self" className="px-6 py-3">Servicios</a>
  <a href="/#subastas"  target="_self" className="px-6 py-3">Subastas</a>
  <a href="/#soporte"   target="_self" className="px-6 py-3">Soporte</a>
  <a href="/#contacto"  target="_self" className="px-6 py-3">Contacto</a>

  {/* 👉 SPA route independiente */}
  <Link href="/marketplace" className="px-6 py-3">Marketplace</Link>
</nav>
target="_self": fuerza recarga completa y evita que Wouter intercepte.

El href debe empezar con / para apuntar siempre a /#contacto, no a /marketplace#contacto.

3️. Verifica los IDs en Home.tsx
Cada sección de la home debe existir con el id exacto:

jsx
Copiar
Editar
export default function Home() {
  return (
    <>
      <section id="inicio">…</section>
      <section id="nosotros">…</section>
      <section id="servicios">…</section>
      <section id="subastas">…</section>
      <section id="soporte">…</section>
      <section id="contacto">…</section>
    </>
  )
}
Si hay un typo en cualquiera de esos id no va a hacer scroll.

Prueba final
Sitio arranca en / → clic “Marketplace” → /marketplace.

Desde /marketplace → clic “Contacto” → debe recargar en /#contacto y bajar al <section id="contacto">.

Igual con cualquier otra sección: “Servicios”, “Subastas”, etc.

Si sigues estos 3 pasos al pie de la letra, no habrá más saltos a “Inicio” ni comportamientos extraños.
